# Dual AI Workflow

Claude Code plugin for dual AI collaboration — Claude Code executes, Cursor Agent reviews.

## Install

```
/plugin marketplace add krislavten/dual-ai-workflow
/plugin install workflow@dual-ai-workflow
```

Then run `/workflow:setup` to configure dependencies and Cursor Agent model.

## Commands

| Command | Description |
|---------|-------------|
| `/workflow:setup` | Install dependencies, select model, configure environment |
| `/workflow:workflow <task> <executor>` | Normal mode — discuss with user, then auto review |
| `/workflow:workflow-yolo <task> <executor>` | YOLO mode — fully automated, user only confirms commit |
| `/workflow:workflow-issue [number]` | Issue-driven mode — task from GitHub Issue, sync via comments |

## How It Works

```
You describe the task
  ↓
Executor (Claude or Cursor) writes proposal
  ↓
Reviewer (the other AI) reviews — up to 5 rounds
  ↓
You approve proposal
  ↓
Executor implements code
  ↓
Reviewer reviews code — up to 5 rounds
  ↓
You approve and commit
```

- **Normal mode**: You participate in design discussion, approve proposal and code
- **YOLO mode**: AI handles everything, you only confirm final commit
- **Issue mode**: Task comes from GitHub Issue, discussion syncs to Issue comments

## Reviewer Configuration

Cursor Agent model and review prompt are configured in `agents/cursor.md` (generated by `/setup`).

Default: `gpt-5.3-codex-xhigh` with strict review prompt — no flattery, finds bugs, checks edge cases.

Override model via env var:
```bash
export WORKFLOW_AGENT_MODEL=opus-4.6-thinking
```

## Issue Mode

When a task has an associated GitHub Issue, all actions sync to Issue comments with identity markers:

- 🧠 **Claude Code** — proposals, implementation summaries
- 🤖 **Cursor Agent** — proposal and code reviews (auto-synced)
- 🔧 **Workflow** — status transitions (auto-synced)

Configure Issue mode env vars (prompted during `/setup`):
```bash
export WORKFLOW_PROJECT_OWNER="your-org"
export WORKFLOW_PROJECT_NUMBER="3"
export WORKFLOW_ISSUE_LABEL="claude-ok"
```
